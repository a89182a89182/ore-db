# ORE 每週流程 (2026-07-04 重建版)

取代舊的 Codex 管線。核心思路:

## 資料
- **單季真實成績** = 相鄰球季 `season-{N}/players.json` 的 career 累計相減(764~現在)
- **當季即時成績** = live 抓的 `season_summary`(777 季起可用)
- 舊管線的致命傷:fantasy 頁面需要登入+只在週日20:00~週一20:00開放,腳本登入
  又被「教練團投票」門檻擋住 → 改用 Chrome(帶登入 session)抓取

## 檔案
- `orelib.py` — 資料載入、career delta 重建、投影模型(投手角色感知)、選隊最佳化
- `backtest.py` — 回測:`python backtest.py 776`
- `tune.py` — 權重搜尋(結果已寫入 orelib.PER_ITEM_W)
- `run_weekly.py` — 產出 HTML 週報:`python run_weekly.py 777`
  - 輸出到 `../reports-weekly/`

## 模型重點
1. 投手投影只採計「同角色」(SP/RP/CP) 的歷史球季 — 角色互換是舊模型 SV 失準主因
2. 各項目權重不同(PER_ITEM_W),由 770-776 七季回測搜尋而得
3. ERA:776 以前的歷史局數資料損壞(全為 0 0/3),用生涯 ERA + 當季 summary ERA 混合排序
4. 菜鳥(無歷史)用同位置群體 40 百分位保守估計,報告中標記 rookie

## 夢幻球隊正式規則(rules/rules.html,選隊演算法已強制執行)
1. 18 人 = 9 野手 + 5 先發 + 3 中繼 + 1 救援
2. 12 支球隊每隊至少 1 人、最多 2 人
3. AVG/ERA 項目需至少 8 名非電腦球員才列入排名(演算法直接排除電腦球員)
4. 同分時平均年薪低者勝(演算法以年薪為次要目標)
5. 各項目冠軍獎金 = 參加人數 x 1000萬 / 8;前10名有安慰獎 500萬
